# Описание файлов данных

## 1. stops.txt

Хранит данные об остановках транспортной системы: уникальный идентификатор остановки и название остановки.

**Формат записи в файл:**

`<id><название>`

**Пример записи в файл:**

```
1|Центральная площадь
2|Железнодорожный вокзал
3|Университет
```

---

## 2. vehicles.txt

Хранит данные о транспортных средствах: тип транспорта, модель и государственный номер.

**Формат записи в файл:**

`<тип_транспорта><модель><госномер>`

**Пример записи в файл:**

```
Автобус|ПАЗ-3205|А123БВ 77
Трамвай|71-623|Т456ГД 77
Троллейбус|ЗиУ-682|В789ЕЖ 77
```

**Примечание:** Тип транспорта может быть: "Автобус", "Трамвай" или "Троллейбус".

---

## 3. drivers.txt

Хранит персональные данные водителей: имя, фамилия, отчество и категория водительских прав.

**Формат записи в файл:**

`<имя><фамилия><отчество><категория>`

**Пример записи в файл:**

```
Иван|Иванов|Иванович|D
Петр|Петров|Петрович|D
Сергей|Сергеев||T
```

**Примечание:** Отчество и категория могут быть пустыми.

---

## 4. routes.txt

Хранит информацию о маршрутах: номер маршрута, тип транспорта, список остановок и дни недели работы.

**Формат записи в файл:**

`<номер><тип_транспорта><остановка1;остановка2;...><день1,день2,...>`

**Пример записи в файл:**

```
1|Автобус|Центральная площадь;Железнодорожный вокзал;Университет|1,2,3,4,5,6,7
2|Трамвай|Вокзал;Площадь Победы;Парк|1,2,3,4,5
```

**Примечание:** 
- Остановки разделяются символом `;` (точка с запятой)
- Дни недели разделяются символом `,` (запятая), где 1 - понедельник, 7 - воскресенье

---

## 5. trips.txt

Хранит данные о рейсах: уникальный идентификатор рейса, данные маршрута, транспортного средства, водителя, время отправления, день недели и расписание прибытия на остановки.

**Формат записи в файл:**

`<id><маршрут><транспорт><водитель><время_отправления><день_недели><расписание>`

**Формат маршрута:** `<номер>|<тип>|<остановки>|<дни>`

**Формат транспорта:** `<тип>|<модель>|<госномер>`

**Формат водителя:** `<имя>|<фамилия>|<отчество>|<категория>`

**Формат времени:** `<ЧЧ:ММ>` (24-часовой формат)

**Формат расписания:** `<остановка1>=<время1>;<остановка2>=<время2>;...`

**Пример записи в файл:**

```
1|1|Автобус|Центральная площадь;Вокзал|1,2,3,4,5,6,7|Автобус|ПАЗ-3205|А123БВ 77|Иван|Иванов|Иванович|D|08:00|1|Центральная площадь=08:00;Вокзал=08:15
```

---

## 6. admins.bin

Хранит учетные записи администраторов системы в бинарном формате.

**Формат:**

- Размер (size_t) - количество учетных записей
- Для каждой записи:
  - Размер (size_t) - длина имени пользователя
  - Строка (char[]) - имя пользователя
  - Размер (size_t) - длина пароля
  - Строка (char[]) - пароль

**Примечание:** Данный файл создается и обновляется программой автоматически. Ручное редактирование не рекомендуется.

---

## Общие замечания

1. Все текстовые файлы используют кодировку UTF-8.
2. Разделителем полей в текстовых файлах является символ `|` (вертикальная черта).
3. Разделителем элементов списка (остановок) является символ `;` (точка с запятой).
4. Разделителем дней недели является символ `,` (запятая).
5. Разделителем пар "остановка=время" в расписании является символ `;` (точка с запятой).
6. Каждая запись в текстовых файлах занимает одну строку.
7. Пустые строки в файлах игнорируются.
8. При наличии дубликатов (по уникальному идентификатору) программа пропускает повторяющиеся записи.
9. Файл `admins.bin` является бинарным и не предназначен для ручного редактирования.
