# Структура входных данных

Входными данными программы являются файлы, содержащие необходимые данные для функционирования программы:

## 1. stops.txt

В данном файле хранятся данные об остановках: уникальный идентификатор остановки, название остановки.

**Формат записи:** `ID|Название`

**Пример:**
```
1|Центральная площадь
2|Железнодорожный вокзал
3|Университет
```

**Поля:**
- Уникальный идентификатор остановки (целое число)
- Название остановки (строка)

---

## 2. vehicles.txt

В данном файле хранятся данные о транспортных средствах: тип транспорта, модель, государственный номер.

**Формат записи:** `Тип|Модель|Госномер`

**Пример:**
```
Автобус|ПАЗ-3205|А123БВ 77
Трамвай|71-623|Т456ГД 77
Троллейбус|ЗиУ-682|В789ЕЖ 77
```

**Поля:**
- Тип транспорта (строка: "Автобус", "Трамвай" или "Троллейбус")
- Модель транспортного средства (строка)
- Государственный номер (строка)

---

## 3. drivers.txt

В данном файле хранятся данные о водителях: имя, фамилия, отчество.

**Формат записи:** `Имя|Фамилия|Отчество`

**Пример:**
```
Иван|Иванов|Иванович
Петр|Петров|Петрович
Сергей|Сергеев|
```

**Поля:**
- Имя водителя (строка)
- Фамилия водителя (строка)
- Отчество водителя (строка, может быть пустым)

---

## 4. routes.txt

В данном файле хранятся данные о маршрутах: номер маршрута, тип транспорта, список остановок через точку с запятой, дни недели работы через запятую.

**Формат записи:** `Номер|ТипТранспорта|Остановка1;Остановка2;Остановка3|День1,День2,День3`

**Пример:**
```
1|Автобус|Центральная площадь;Железнодорожный вокзал;Университет|1,2,3,4,5,6,7
2|Трамвай|Вокзал;Площадь Победы;Парк|1,2,3,4,5
```

**Поля:**
- Номер маршрута (целое число)
- Тип транспорта (строка: "Автобус", "Трамвай" или "Троллейбус")
- Список остановок (строки, разделенные точкой с запятой)
- Дни недели работы (целые числа от 1 до 7, разделенные запятыми, где 1 - понедельник, 7 - воскресенье)

---

## 5. trips.txt

В данном файле хранятся данные о рейсах: уникальный идентификатор рейса, данные маршрута, данные транспортного средства, данные водителя, время отправления, день недели, расписание прибытия на остановки.

**Формат записи:** `ID|Маршрут|Транспорт|Водитель|ВремяОтправления|ДеньНедели|Расписание`

**Формат маршрута:** `Номер|ТипТранспорта|Остановка1;Остановка2|Дни`

**Формат транспорта:** `Тип|Модель|Госномер`

**Формат водителя:** `Имя|Фамилия|Отчество`

**Формат времени:** `ЧЧ:ММ` (24-часовой формат)

**Формат расписания:** `Остановка1=Время1;Остановка2=Время2`

**Пример:**
```
1|1|Автобус|Центральная площадь;Вокзал|1,2,3,4,5,6,7|Автобус|ПАЗ-3205|А123БВ 77|Иван|Иванов|Иванович|08:00|1|Центральная площадь=08:00;Вокзал=08:15
```

**Поля:**
- Уникальный идентификатор рейса (целое число)
- Данные маршрута (номер, тип транспорта, остановки, дни недели)
- Данные транспортного средства (тип, модель, госномер)
- Данные водителя (имя, фамилия, отчество)
- Время отправления (строка в формате ЧЧ:ММ)
- День недели (целое число от 1 до 7)
- Расписание прибытия на остановки (пары "остановка=время", разделенные точкой с запятой)

---

## 6. admins.bin

В данном файле хранятся данные об учетных записях администраторов в бинарном формате: количество записей, для каждой записи - длина имени пользователя, имя пользователя, длина пароля, пароль.

**Формат:**
- Размер (size_t) - количество учетных записей
- Для каждой записи:
  - Размер (size_t) - длина имени пользователя
  - Строка (char[]) - имя пользователя
  - Размер (size_t) - длина пароля
  - Строка (char[]) - пароль

**Поля:**
- Количество учетных записей (size_t)
- Имя пользователя (строка переменной длины)
- Пароль (строка переменной длины)

**Примечание:** Данный файл создается и обновляется программой автоматически. Ручное редактирование не рекомендуется.

---

## Общие замечания

1. Все текстовые файлы используют кодировку UTF-8.
2. Разделителем полей в текстовых файлах является символ `|` (вертикальная черта).
3. Разделителем элементов списка является символ `;` (точка с запятой).
4. Разделителем дней недели является символ `,` (запятая).
5. Каждая запись в текстовых файлах занимает одну строку.
6. Пустые строки в файлах игнорируются.
7. При наличии дубликатов (по уникальному идентификатору) программа пропускает повторяющиеся записи.
8. Файл `admins.bin` является бинарным и не предназначен для ручного редактирования.
